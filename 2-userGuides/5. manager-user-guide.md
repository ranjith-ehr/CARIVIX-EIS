# Manager User Guide

This guide covers the manager-specific features of the Attendance Tracking system — including team monitoring, anomaly review, predictive reports, and chatbot queries for team-level data.

---

## Manager Dashboard Overview

After login, managers land on the **Manager Dashboard**, which provides a team-level view of attendance and performance data.

| Dashboard Panel | What It Shows |
|---|---|
| **Team Attendance Summary** | Today's present / absent / late count across your team |
| **Anomaly Alerts** | Active anomaly flags requiring review |
| **Performance Distribution** | Score breakdown across team members |
| **Attendance Trend** | 30-day attendance rate trend for the team |
| **Forecast Panel** | Predicted attendance levels for the next 7 days |

---

## Viewing Team Attendance

1. Go to **Team → Attendance** from the left navigation.
2. The table shows all team members with today's status by default.
3. Use the **date range picker** to filter by day, week, or custom range.
4. Use the **Status filter** to show only Absent, Late, or flagged employees.
5. Click any employee row to view their individual attendance detail — including login/logout times, GPS location, work hours, and face verification result.

### Exporting Attendance Data
1. Apply any filters you need (date range, status, department).
2. Click **Export** → choose **CSV** or **PDF**.
3. The file downloads immediately or is emailed to you for large date ranges.

## Reviewing Anomaly Alerts

Anomalies are flagged automatically by the system's Isolation Forest model when attendance behavior falls outside expected patterns.

### Accessing Alerts
1. Go to **Alerts → Anomaly Flags** or click the alert count on the dashboard.
2. Each alert shows: Employee name, anomaly type, date/time, severity, and current status.

### Anomaly Types

| Type | Description |
|---|---|
| Sudden Absenteeism | Unexpected consecutive absences after a consistent attendance pattern |
| Suspicious Login Pattern | Login times that deviate significantly from the employee's baseline |
| Unrealistic Work Hours | Sessions with abnormally short or long durations |
| Location Mismatch | Check-in from outside the approved geofence zone |
| Simultaneous Location | Same employee appearing in two locations within an impossible time window |

### Resolving an Alert
1. Click the alert to open the detail view.
2. Review the flagged session data.
3. Select one of the following actions:
   - **Acknowledge** — Mark as reviewed with no action required.
   - **Escalate** — Forward to HR or Admin for further action.
   - **Dismiss as False Positive** — Mark as a system error; this feeds back into model improvement.
4. Add a note before submitting (required for Escalate and Dismiss actions).

## Accessing Predictive Reports

1. Go to **Reports → Predictive Analytics**.
2. Select the report type:

| Report Type | Description |
|---|---|
| **Attendance Forecast** | Predicted daily attendance count for the next 7 days |
| **Performance Forecast** | Estimated performance score changes per employee |
| **Employee Classification** | Current classification of each team member (Consistent / Irregular / High-Risk) |
| **Absenteeism Risk** | Employees with a high predicted probability of future absence |

3. Apply filters (department, date range, classification) and click **Generate**.
4. Export reports as CSV or PDF as needed.

---

## Generating and Exporting Team Reports

1. Go to **Reports → Team Reports**.
2. Choose a report template:
   - Daily Attendance Summary
   - Weekly Work Hours Report
   - Monthly Performance Report
   - Anomaly Summary Report
3. Set the date range and click **Generate Report**.
4. Preview the report in-browser or click **Export** to download.

### Scheduling Automated Reports
1. From the report preview, click **Schedule**.
2. Set frequency: Daily, Weekly, or Monthly.
3. Add recipient email addresses.
4. Choose format: CSV or PDF.
5. Click **Save Schedule**. Reports are delivered at 08:00 local time on the configured days.

## Using the Chatbot for Team Queries

The chatbot (bottom-right chat icon) supports team-level queries for managers.

### Example Manager Queries

- "Who was absent today?"
- "Show me the attendance rate for my team this month."
- "Which employees are classified as High-Risk?"
- "Summarize anomaly alerts from last week."
- "What is the predicted attendance for next Monday?"
- "List employees with the lowest performance scores."

### Tips
- The chatbot automatically scopes queries to your team. You do not need to specify department names.
- For individual employee queries, include the employee's name: "Show work hours for [Name] this week."
- If a query returns too much data, ask it to narrow down: "Show only the bottom 5."

## Configuring Alert Thresholds

Managers can adjust alert sensitivity for their team.

1. Go to **Settings → Alert Configuration**.
2. Adjust thresholds per alert type:

| Setting | Default | Description |
|---|---|---|
| Anomaly Sensitivity | Medium | Low / Medium / High — controls how aggressively the model flags behaviors |
| Absence Alert Trigger | 2 consecutive days | Number of consecutive absences before an alert fires |
| Late Login Alert | After configured threshold | Set by admin; manager can increase but not decrease |

3. Click **Save**. Changes apply to your team only.
