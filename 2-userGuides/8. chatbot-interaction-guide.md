# Chatbot Interaction Guide

The AI chatbot allows employees and managers to query attendance data, performance metrics, and task summaries using natural language — without navigating through reports manually.

The chatbot is accessible from the **chat bubble icon** on the bottom right of any page, on both web and mobile.

---

## How the Chatbot Works

When you type a query, the chatbot:

1. **Detects your intent** — Understands what type of information you are asking for.
2. **Extracts entities** — Identifies key details in your query such as dates, employee names, or metric names.
3. **Retrieves relevant data** — Fetches your attendance or performance records from the system.
4. **Generates a response** — Returns a plain-language answer, a data table, or a reference to a dashboard chart.

The chatbot is scoped to your account by default. Managers can query team-level data by specifying a team or employee name.

## What Employees Can Ask

### Attendance

| Query Example | What It Returns |
|---|---|
| "What is my attendance percentage this month?" | Attendance rate for the current month |
| "How many days was I absent in March?" | Absence count for the specified month |
| "Was I marked late this week?" | Late login records for the current week |
| "Show my attendance for the last 30 days." | Day-by-day attendance table |
| "What was my check-in time yesterday?" | Login timestamp for the previous day |

### Work Hours

| Query Example | What It Returns |
|---|---|
| "Show my work hours this week." | Daily work hours for Mon–today |
| "What was my total work time in February?" | Aggregate work hours for the month |
| "Did I work overtime yesterday?" | Yesterday's session duration vs. standard hours |
| "How many hours did I work on Monday?" | Work hours for a specific day |

### Performance

| Query Example | What It Returns |
|---|---|
| "What is my current performance score?" | Current score with classification |
| "Why did my score drop this week?" | Top contributing factors (SHAP-based explanation) |
| "What is my attendance consistency score?" | Consistency score component only |

### Tasks

| Query Example | What It Returns |
|---|---|
| "Summarize my tasks from yesterday." | Task log summary for the previous day |
| "How many tasks did I complete this week?" | Task count for the current week |

## What Managers Can Ask

Managers can query their own data using the same employee queries above, and additionally ask about their team.

### Team Attendance

| Query Example | What It Returns |
|---|---|
| "Who was absent today?" | List of absent team members |
| "What is the team attendance rate this month?" | Aggregate attendance rate for the team |
| "Show me all late logins from last week." | Late login records across the team |
| "How many employees were present yesterday?" | Count of present team members |

### Anomalies and Alerts

| Query Example | What It Returns |
|---|---|
| "Summarize anomaly alerts from last week." | List of flagged alerts with type and employee |
| "Are there any unresolved anomaly flags?" | Count and list of open alerts |
| "Which employees have location mismatch flags?" | Employees with active GIS anomalies |

### Performance and Classification

| Query Example | What It Returns |
|---|---|
| "List employees classified as High-Risk." | High-Risk employee list with scores |
| "Who has the lowest performance score on my team?" | Bottom performers with scores |
| "Show performance trends for [Employee Name]." | Score trend over the last 30 days |

### Predictions

| Query Example | What It Returns |
|---|---|
| "What is the predicted attendance for next Monday?" | Forecast figure with confidence range |
| "Which employees are at risk of absenteeism next week?" | High-risk forecast list |

## Phrasing Tips

**Include a time reference** when asking about a specific period:
- Good: "Show my work hours for last week."
- Vague: "Show my work hours." *(defaults to today)*

**Use "my" for personal queries and the employee's name for manager queries:**
- Employee: "What is my attendance rate?"
- Manager: "What is Sarah's attendance rate this month?"

**Ask one question at a time** for best results. The chatbot handles compound queries but single-focus questions return faster and more accurate responses.

**Be specific about metrics** — use the metric name as shown in your dashboard:
- "attendance percentage", "performance score", "work hours", "late login count"

## Response Types

| Response Format | When It Appears |
|---|---|
| **Plain text answer** | Single-value queries (e.g., "What is my score?") |
| **Data table** | List queries (e.g., "Show my attendance for last month") |
| **Chart reference** | Trend queries — the chatbot links to the relevant dashboard panel |
| **Clarification prompt** | When the query is ambiguous; the chatbot asks a follow-up question |
| **Out-of-scope message** | When the query is outside what the chatbot can answer |

---

## Fallback and Out-of-Scope Behavior

The chatbot will tell you when it cannot answer a query. Common out-of-scope examples:

- Payroll, salary, or leave balance information — these are handled by separate systems.
- Queries about other employees (for non-manager accounts).
- Future event scheduling or task creation.
- Anything unrelated to attendance, performance, or task data.

When a query falls outside scope, the chatbot responds with a message such as:
> *"I can't retrieve that information. You may find it in [suggested section] or by contacting your HR team."*

---

## Known Limitations

- The chatbot does not retain memory between separate sessions. Each conversation starts fresh.
- Queries referencing periods more than **12 months** in the past may return incomplete data.
- Entity extraction for non-English names may occasionally require exact spelling.
- Real-time data (e.g., "Is [employee] checked in right now?") reflects a sync delay of up to 60 seconds.
