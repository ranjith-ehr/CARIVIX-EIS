# Release Notes

Version history for the AI-Based Attendance Tracking and Employee Management System. Releases are listed in reverse chronological order.

Each release entry covers: new features, improvements, bug fixes, breaking changes, and migration notes where applicable.

---

## v1.2.0 — 2026-04-11

### New Features
- **Scheduled Reports** — Managers and admins can now configure attendance and performance reports for automated recurring delivery via email. Supports Daily, Weekly, and Monthly frequencies with CSV or PDF output.
- **Score Breakdown Panel** — Employees can now view a SHAP-based plain-language explanation of their performance score directly from the dashboard. Accessible via **Score Breakdown** on the Performance widget.
- **Absenteeism Risk Report** — New predictive report listing employees with a high predicted probability of future absence, available under **Reports → 

### Improvements
- Attendance trend forecast horizon extended from 5 days to 7 days.
- Chatbot now supports manager queries for team-level anomaly summaries and classification data.
- Mobile app GPS lock time reduced by approximately 30% through improved location caching.
- Anomaly alert emails now include a direct deep link to the alert detail view.

### Bug Fixes
- Fixed an issue where sessions with logout before midnight and login after midnight on the same calendar day were calculated as negative work hours.
- Fixed duplicate anomaly alerts being generated for the same flagged session when the model retraining job ran while an existing alert was in the **Acknowledged** state.
- Fixed incorrect attendance rate displayed on the employee dashboard when the month contains a public holiday that was added to the system after the month began.
- Fixed mobile app crash on iOS 15 when camera permission was revoked mid-session.

### Deprecations
- The `GET /attendance/summary` endpoint is deprecated. Use `GET /attendance/me` with date range parameters instead. The deprecated endpoint will be removed in v1.4.0.

## v1.1.0 — 2026-02-28

### New Features
- **Employee Classification** — The system now classifies each employee as Consistent, Irregular, or High-Risk based on attendance patterns. Classification is visible on the manager dashboard and in the Monthly Performance Report.
- **Chatbot — Employee Queries** — AI chatbot now available for employees to query personal attendance data, work hours, and performance scores in natural language.
- **Manual Attendance Correction** — Managers can now correct login and logout times for incomplete or missing sessions directly from the Manager Dashboard. Corrections are logged in the audit trail.
- **Audit Log Export** — Admins can export filtered audit log data as CSV from the Admin Console.

### Improvements
- Performance score now uses a 30-day rolling window (previously recalculated from the start of the calendar month).
- Anomaly detection model retrained with an expanded 90-day training window, reducing false positives by approximately 18%.
- Admin Console geofence map now supports satellite view for more accurate polygon boundary placement.
- API rate limit increased from 30 to 60 requests per minute per user.

### Bug Fixes
- Fixed an issue where employees with no check-out record were not appearing in the Absent filter on the team attendance view.
- Fixed incorrect late login count in the Weekly Work Hours Report when the reporting period crossed a month boundary.
- Fixed face enrollment failure not generating an admin notification after 3 consecutive failed attempts.
- Fixed timezone offset error causing check-in timestamps to display in UTC instead of local time on the employee dashboard.

### Breaking Changes
- **Authentication endpoint path changed.** `/api/login` is renamed to `/auth/login`. Clients must update their integration before upgrading. The old path returns `404` from this version.
- **Attendance record response schema updated.** The `check_in` and `check_out` fields are renamed to `login_time` and `logout_time` in all attendance API responses. Update any client parsing these fields.

### Migration Notes
1. Update all API client integrations to use `/auth/login` in place of `/api/login`.
2. Update field references from `check_in` / `check_out` to `login_time` / `logout_time` in all attendance response parsers.
3. Re-run any scheduled exports or downstream pipelines that consume attendance API data to validate against the updated schema.

## v1.0.1 — 2026-01-20

### Bug Fixes
- Fixed GPS validation returning a false failure for employees near the edge of a polygon geofence zone when the device GPS reading had sub-10-meter variance.
- Fixed face verification confidence score not being written to the attendance record, causing null values in the performance pipeline.
- Fixed mobile app notification not delivered when the device was in Do Not Disturb mode with no app exception configured.
- Fixed admin role not being able to access the Audit Log page despite having the correct permissions (UI routing error).

## v1.0.0 — 2026-01-05

### Initial Release

Core platform launch including:

- Attendance check-in and check-out with face verification and GPS validation.
- Attendance status calculation: Present, Late, Half-Day, Absent.
- Work hour calculation and idle time detection.
- Performance scoring (composite metric: attendance rate, work hours, late logins, idle time, overtime).
- Anomaly detection via Isolation Forest model.
- Performance prediction via XGBoost regression model.
- Attendance trend forecasting via time-series model.
- Employee personal dashboard and manager team dashboard.
- Daily, Weekly, and Monthly report generation with CSV and PDF export.
- REST API with JWT authentication across four blueprints: session, attendance, task, activity.
- React Native mobile app (iOS and Android) with GPS check-in, face verification, and push notifications.
- Admin Console: user management, geofence configuration, attendance rule configuration.
- Cloud deployment on Google Cloud Platform: Cloud Run, Cloud SQL, Memorystore, GPU Compute Engine.
- CI/CD pipeline: GitHub Actions → Docker → Artifact Registry → Terraform → Cloud Run.

## Versioning Policy

This system follows [Semantic Versioning](https://semver.org/):

| Version Segment | When It Increments |
|---|---|
| **Major (X.0.0)** | Breaking changes that require client-side migration |
| **Minor (0.X.0)** | New features added in a backward-compatible manner |
| **Patch (0.0.X)** | Backward-compatible bug fixes |

Breaking changes are always documented with migration notes in the release entry.
