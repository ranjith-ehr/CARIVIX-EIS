# Glossary

Definitions for system-specific terms, ML/AI concepts, and technical terminology used across the Knowledge Base. Terms are listed alphabetically.

---

## A

**Anomaly**
An attendance event that deviates significantly from an employee's established behavioral baseline. Detected automatically by the Isolation Forest model and routed to the employee's manager as an alert.

**Anomaly Score**
A numerical value (0–1) assigned by the Isolation Forest model to each attendance record. Higher scores indicate greater deviation from normal behavior. Records above a configured threshold are flagged as anomalies.

**Attendance Rate**
The percentage of working days an employee was present or late, calculated as:
`(Days Present / Total Working Days) × 100`
Excludes weekends and public holidays as configured by the administrator.

**Attendance Consistency Score**
A feature engineered from attendance history that measures how regularly and predictably an employee logs in. High consistency means low variance in login times and few unexpected absences.

**Attendance Status**
The classification assigned to an employee for each working day: **Present**, **Late**, **Half-Day**, or **Absent**. Determined by configured attendance rules applied to the day's check-in and checkout data.

**Attendance Streak**
The number of consecutive working days an employee has been present. Used as an input feature for performance prediction and classification models.

---

## B

**Behavioral Baseline**
The pattern of normal attendance behavior established for an individual employee based on their historical data. The anomaly detection model measures deviations relative to this baseline rather than applying global fixed thresholds.

**Buffer Tolerance**
An additional margin (in meters) added to a geofence boundary to account for GPS signal drift, particularly indoors. Configured per geofence zone by the administrator. Default value: 20 meters.

---

## C

**Check-In**
The action of recording the start of an employee's workday. Requires face verification and GPS validation to succeed.

**Check-Out**
The action of recording the end of an employee's workday. Requires face verification. Work hours for the day are calculated upon successful check-out.

**Classification (Employee)**
A category assigned to each employee by the Logistic Regression model based on their attendance patterns: **Consistent**, **Irregular**, or **High-Risk**.

**Contamination Parameter**
A setting in the Isolation Forest model that specifies the expected proportion of anomalous records in the dataset. It controls the anomaly score threshold at which records are flagged. A higher contamination value makes the model flag more records.

**Cross-Validation**
A model evaluation technique that splits training data into multiple folds, training and evaluating the model on different subsets to ensure the model generalizes well to unseen data rather than overfitting to the training set.

---

## D

**DistilBERT**
A lightweight variant of the BERT language model used in the chatbot's intent detection component. It classifies the user's query into a predefined intent category (e.g., "attendance query", "work hours query") to route the request appropriately.

---

## E

**Embedding (Facial)**
A numerical vector — a list of floating-point numbers — that represents the unique geometric features of a person's face as extracted by the facial recognition model. The system stores embeddings rather than raw photos. Two embeddings are compared using a similarity score to verify identity.

**Entity Extraction**
See **Named Entity Recognition (NER)**.

---

## F

**FAISS**
Facebook AI Similarity Search — a library used in the chatbot's RAG pipeline to index and retrieve relevant documents or data snippets from a vector store based on semantic similarity to the user's query.

**Feature Engineering**
The process of creating new input variables (features) from raw data to improve ML model performance. Examples in this system include: weekly attendance averages, login time deviation, attendance streaks, and work hour variance.

**Facial Embedding**
See **Embedding (Facial)**.

**Forecast Horizon**
The number of future time periods a predictive model generates predictions for. In this system, the attendance trend forecast has a horizon of 7 days and the performance score forecast covers 30 days.

---

## G

**Geofence**
A virtual boundary drawn around a physical location — defined as a radius around a centre point or a custom polygon. Employees must be within an assigned geofence zone for their check-in location to be validated.

**GIS (Geographic Information System)**
A framework for capturing, analyzing, and validating location data. In this system, GIS refers to the module that compares employee GPS coordinates against geofence zone boundaries and performs location anomaly checks.

**GPS Drift**
Small inaccuracies in GPS location readings caused by signal interference, building materials, or atmospheric conditions. Indoor environments typically experience 10–50 meter drift. The system applies a buffer tolerance to account for this.

---

## H

**Hyperparameter Tuning**
The process of finding the optimal configuration settings for an ML model (e.g., learning rate, tree depth, regularization strength) that maximize performance on validation data. Performed using grid search or Bayesian optimization.

---

## I

**Idle Time**
Periods during a working session where no activity event is recorded. Calculated by comparing the timestamps of consecutive activity heartbeats. Gaps exceeding the configured idle time threshold (default: 30 minutes) are classified as idle periods.

**Intent Detection**
The component of the chatbot pipeline that classifies the user's natural language query into a predefined intent category. Powered by DistilBERT or RoBERTa. The detected intent routes the query to the appropriate data retrieval logic.

**Isolation Forest**
An unsupervised machine learning algorithm used for anomaly detection. It works by randomly partitioning data features and measuring how quickly individual data points can be isolated from the rest. Anomalous records are isolated in fewer partitions (shorter paths) and receive higher anomaly scores.

---

## J

**JWT (JSON Web Token)**
A compact, digitally signed token used for authentication. The system issues a JWT upon login. The token encodes the user's identity and role and must be included in the `Authorization: Bearer` header of all API requests. Tokens expire after 8 hours.

---

## L

**LangChain**
An open-source framework used to orchestrate the chatbot's Retrieval Augmented Generation (RAG) pipeline. It connects the intent detection output, the vector store retrieval step, and the LLM response generation into a single coordinated workflow.

**Late Login**
A check-in recorded after the configured work start time plus the late login threshold. For example, if work starts at 09:00 and the threshold is 15 minutes, any login after 09:15 is a late login. Affects attendance status and performance score.

**LLM (Large Language Model)**
The AI model responsible for generating natural language responses in the chatbot. The system uses **LLaMA 3** or **Mistral**, hosted on a GPU Compute Engine VM.

**Login Time Deviation**
A feature measuring how much an employee's actual login time varies from their historical average login time. High deviation is a signal used in anomaly detection and performance modeling.

---

## M

**Min-Max Scaling**
A data normalization technique that rescales feature values to a fixed range (typically 0–1) by subtracting the minimum value and dividing by the range. Used in the data pipeline before ML model training to ensure features with different units are comparable.

**ML-Ready Dataset**
The processed and formatted dataset output by the data pipeline, ready for consumption by machine learning models. It has been cleaned, transformed, feature-engineered, and normalized.

---

## N

**Named Entity Recognition (NER)**
A natural language processing technique that identifies and extracts specific entities from text — such as dates, employee names, or task IDs — from the user's chatbot query. Powered by **spaCy** in this system.

---

## O

**Overtime**
Any session where the employee's work hours exceed the configured standard work day duration. Overtime frequency is a positive contributor to the performance score. Overtime sessions on weekends or public holidays are recorded separately.

---

## P

**Performance Score**
A composite metric (0–100) measuring an employee's attendance behavior over a 30-day rolling window. Calculated from: attendance rate, average work hours, late login count, idle time ratio, and overtime frequency.

**Pinecone**
A managed vector database used as an alternative to FAISS for the chatbot's RAG pipeline. Stores and retrieves document embeddings for context retrieval before LLM response generation.

**Predictive Analytics**
The use of ML models to generate forecasts and classifications about future behavior. In this system: performance score prediction (XGBoost), employee classification (Logistic Regression), and attendance trend forecasting (time-series model).

---

## R

**RAG (Retrieval Augmented Generation)**
A chatbot architecture pattern where relevant documents or data records are retrieved from a knowledge base before the LLM generates its response. This grounds the response in actual system data rather than relying solely on the model's internal knowledge.

**RoBERTa**
A transformer-based language model used as an alternative to DistilBERT for intent detection in the chatbot. Provides higher accuracy at the cost of slightly higher inference latency.

**Rolling Window**
A time window that moves forward with each passing day, always covering a fixed period relative to today. The performance score uses a 30-day rolling window — always reflecting the most recent 30 days of behavior.

---

## S

**SHAP (SHapley Additive exPlanations)**
A model explainability method that calculates how much each input feature contributed to a specific prediction. In this system, SHAP values power the plain-language score breakdown shown to employees on their dashboard.

**SMOTE (Synthetic Minority Over-sampling Technique)**
A technique for handling imbalanced datasets in ML training. Generates synthetic samples of the underrepresented class (e.g., anomalous records) to prevent the model from being biased toward the majority class (normal records).

**spaCy**
An open-source NLP library used for Named Entity Recognition in the chatbot pipeline. Extracts dates, employee names, and task identifiers from user queries to support data retrieval.

---

## W

**Work Hour Variance**
A feature measuring the day-to-day variability in an employee's recorded work hours. High variance may indicate irregular scheduling, frequent early departures, or inconsistent task engagement. Used as an input to performance and anomaly models.

---

## X

**XGBoost (Extreme Gradient Boosting)**
A high-performance gradient boosting ML framework used for the performance score prediction model. It builds an ensemble of decision trees sequentially, with each tree correcting the errors of the previous one.

---

## Z

**Zone Assignment**
The configuration that links a geofence zone to specific employees or departments. An employee can be assigned to multiple zones and can check in from any assigned zone.
