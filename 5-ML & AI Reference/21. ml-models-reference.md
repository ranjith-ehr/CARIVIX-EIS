# ML Models Reference

This document is the authoritative reference for all machine learning models in the Attendance Tracking system. It covers each model's purpose, input features, output format, framework, evaluation metrics, and artifact storage location.

---

## Model Inventory

| Model | Type | Purpose | Framework | Retraining Frequency |
|---|---|---|---|---|
| Performance Predictor | Regression | Predict future performance score (30-day forecast) | XGBoost | Monthly |
| Employee Classifier | Classification | Classify employees as Consistent / Irregular / High-Risk | Logistic Regression (Scikit-learn) | Monthly |
| Anomaly Detector | Anomaly Detection | Flag abnormal attendance records | Isolation Forest (Scikit-learn) | Bi-weekly |
| Attendance Forecaster | Time-Series | Forecast team attendance headcount (7-day horizon) | Prophet / ARIMA | Weekly |

## Model 1 — Performance Predictor (XGBoost)

### Purpose
Estimates each employee's performance score for the next 30 days based on their behavioral history.

### Input Features

| Feature | Type | Description |
|---|---|---|
| `attendance_rate_30d` | float | Rolling 30-day attendance rate (0–100) |
| `attendance_rate_90d` | float | Rolling 90-day attendance rate (0–100) |
| `avg_work_hours_30d` | float | Rolling 30-day average daily work hours |
| `late_login_count_30d` | integer | Count of late logins in the last 30 days |
| `idle_ratio_30d` | float | Idle time / total session time (last 30 days) |
| `overtime_frequency_30d` | float | Proportion of sessions with overtime (last 30 days) |
| `attendance_consistency_score` | float | Std-based consistency metric (0–1; higher = more consistent) |
| `login_time_deviation` | float | Std of login hour from employee's personal mean |
| `attendance_streak` | integer | Current consecutive present days |
| `work_hour_variance_14d` | float | Variance of work hours over the last 14 days |

All features are Min-Max scaled to [0, 1] before model input.

### Output

```python
{
    "employee_id": "usr_a1b2c3",
    "predicted_scores": [
        {"date": "2026-04-12", "score": 71.4, "lower_bound": 65.2, "upper_bound": 77.6},
        {"date": "2026-04-13", "score": 72.1, "lower_bound": 66.0, "upper_bound": 78.2},
        ...  # 30 days
    ],
    "model_version": "xgb_v3"
}
```

### Evaluation Metrics

| Metric | Target | Last Measured Value |
|---|---|---|
| RMSE (Root Mean Squared Error) | < 8.0 | 6.3 |
| MAE (Mean Absolute Error) | < 6.0 | 4.9 |
| R² | > 0.75 | 0.81 |

Metrics are evaluated on a held-out test set (last 20% of chronologically ordered data).

### Artifact Storage

```
gs://attendance-ml-artifacts/performance-predictor/
    ├── xgb_v3.pkl              # Trained model
    ├── xgb_v3_metadata.json    # Training date, feature list, metrics
    └── minmax_scaler_v3.pkl    # Fitted scaler for inference-time normalization
```

### Inference Call

```python
import joblib
import pandas as pd

model = joblib.load('xgb_v3.pkl')
scaler = joblib.load('minmax_scaler_v3.pkl')

features = pd.DataFrame([employee_feature_vector])
scaled = scaler.transform(features)
prediction = model.predict(scaled)
```

## Model 2 — Employee Classifier (Logistic Regression)

### Purpose
Assigns each employee to one of three behavioral classification tiers based on their attendance patterns.

### Classes

| Class | Label | Description |
|---|---|---|
| 0 | `consistent` | Regular, predictable attendance within expected parameters |
| 1 | `irregular` | Variable attendance with frequent late logins or inconsistent hours |
| 2 | `high_risk` | Patterns indicating a significant risk of continued underperformance or absenteeism |

### Input Features

Same feature set as the Performance Predictor, with the addition of:

| Feature | Type | Description |
|---|---|---|
| `absence_rate_90d` | float | Proportion of absent days in the last 90 days |
| `anomaly_count_30d` | integer | Number of anomaly flags in the last 30 days |
| `score_trend_direction` | integer | -1 (declining), 0 (stable), +1 (improving); derived from last 30-day score slope |

### Output

```python
{
    "employee_id": "usr_a1b2c3",
    "classification": "irregular",
    "probabilities": {
        "consistent": 0.18,
        "irregular": 0.67,
        "high_risk": 0.15
    },
    "model_version": "lr_v2"
}
```

### Decision Threshold

The default classification uses `argmax(probabilities)`. The threshold for flagging `high_risk` can be lowered independently:

```python
# In config: HIGH_RISK_THRESHOLD = 0.35
# If P(high_risk) >= 0.35, override to high_risk even if not argmax
```

This is configurable in `ml/config/classifier_config.yaml`.

### Evaluation Metrics

| Metric | Target | Last Measured Value |
|---|---|---|
| Weighted F1 Score | > 0.80 | 0.84 |
| Precision (high_risk) | > 0.75 | 0.78 |
| Recall (high_risk) | > 0.70 | 0.73 |
| ROC-AUC (macro) | > 0.85 | 0.88 |

### Class Imbalance Handling

High-risk employees typically represent 10–15% of the workforce. The classifier uses `class_weight='balanced'` in Scikit-learn to compensate:

```python
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(
    class_weight='balanced',
    max_iter=1000,
    C=0.5,         # regularization strength (tuned via GridSearchCV)
    solver='lbfgs'
)
```

### Artifact Storage

```
gs://attendance-ml-artifacts/employee-classifier/
    ├── lr_v2.pkl
    ├── lr_v2_metadata.json
    └── label_encoder_v2.pkl
```

## Model 3 — Anomaly Detector (Isolation Forest)

### Purpose
Assigns an anomaly score to each attendance record and flags records above a threshold as anomalous.

### Input Features

| Feature | Type | Description |
|---|---|---|
| `work_hours` | float | Session work hours |
| `login_hour` | float | Hour of day (0–23) of login |
| `login_time_deviation` | float | Deviation from employee's mean login hour |
| `attendance_streak` | integer | Consecutive present days before this session |
| `idle_ratio` | float | Idle time ratio for this session |
| `gis_validated` | integer | 1 if GIS passed, 0 if failed |
| `face_confidence` | float | Face match confidence score |

Note: Isolation Forest is unsupervised — there are no class labels. Features describe the record itself, not a rolling window.

### Output

```python
{
    "record_id": "att_001abc",
    "anomaly_score": 0.73,       # 0 = normal, 1 = maximally anomalous
    "is_anomaly": True,           # score >= contamination threshold
    "model_version": "iforest_v4"
}
```

### Key Hyperparameters

| Parameter | Value | Notes |
|---|---|---|
| `n_estimators` | 200 | Number of isolation trees |
| `contamination` | 0.05 | Expected proportion of anomalies (5%); tunable in config |
| `max_samples` | `'auto'` | Uses min(256, n_samples) |
| `random_state` | 42 | For reproducibility |

```python
from sklearn.ensemble import IsolationForest

model = IsolationForest(
    n_estimators=200,
    contamination=0.05,
    max_samples='auto',
    random_state=42
)
```

### Contamination Tuning

The `contamination` parameter controls the anomaly score threshold. To adjust:

1. Edit `ml/config/anomaly_config.yaml`: `contamination: 0.05`
2. Trigger a manual retraining job.
3. Monitor false positive rate over the next 7 days.

Lower contamination → fewer flags, lower FP rate. Higher contamination → more flags, higher sensitivity.

### Evaluation

Isolation Forest is unsupervised, so standard labeled metrics do not apply. Evaluation uses:

| Method | Target |
|---|---|
| False positive rate (dismissed alerts / total alerts) | < 15% |
| Coverage of known fraudulent check-ins (injected test cases) | > 85% |
| Precision on labeled anomaly test set | > 0.72 |

### Artifact Storage

```
gs://attendance-ml-artifacts/anomaly-detector/
    ├── iforest_v4.pkl
    └── iforest_v4_metadata.json
```

## Model 4 — Attendance Forecaster (Prophet / ARIMA)

### Purpose
Forecasts daily team attendance headcount for the next 7 days to support workforce planning.

### Model Selection

| Condition | Model Used |
|---|---|
| Sufficient history (> 90 days, stable seasonality) | **Prophet** (Facebook) |
| Insufficient history or high irregularity | **ARIMA** (fallback) |

The selection is made automatically at retraining time based on AIC score comparison.

### Input Data

Daily aggregate attendance series per team (or organization):

```python
# Input DataFrame schema for Prophet
df = pd.DataFrame({
    'ds': pd.date_range('2025-01-01', periods=N, freq='B'),  # business days
    'y':  [float, ...]   # daily attendance headcount or rate
})
```

### Prophet Configuration

```python
from prophet import Prophet

model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False,
    seasonality_mode='multiplicative',
    changepoint_prior_scale=0.05
)
model.add_country_holidays(country_name='IN')  # India public holidays
model.fit(df)

future = model.make_future_dataframe(periods=7, freq='B')
forecast = model.predict(future)
```

### Output

```python
{
    "team_id": "dept_engineering",
    "forecast": [
        {
            "date": "2026-04-14",
            "predicted_headcount": 18,
            "lower_bound": 15,
            "upper_bound": 21,
            "confidence_level": 0.80
        },
        ...  # 7 days
    ],
    "model_type": "prophet",
    "model_version": "forecast_v5"
}
```

### Evaluation Metrics

| Metric | Target | Last Measured Value |
|---|---|---|
| MAPE (Mean Absolute Percentage Error) | < 10% | 7.2% |
| MAE | < 2 headcount | 1.6 |

Evaluation is performed using walk-forward validation on the last 30 days of historical data.

### Artifact Storage

```
gs://attendance-ml-artifacts/attendance-forecaster/
    ├── forecast_v5_prophet.pkl       # or forecast_v5_arima.pkl
    ├── forecast_v5_metadata.json
    └── forecast_v5_team_index.json   # maps team_id to model instance
```

## Model Artifact Versioning Convention

```
{model_type}_v{N}.pkl

Examples:
  xgb_v3.pkl
  lr_v2.pkl
  iforest_v4.pkl
  forecast_v5_prophet.pkl
```

Version `N` increments on each retraining that produces a model that passes evaluation thresholds. A retraining that does not improve on the current production model does not replace it.

The `metadata.json` file for each artifact contains:

```json
{
    "model_type": "xgboost_regressor",
    "version": "xgb_v3",
    "trained_at": "2026-03-01T22:00:00Z",
    "training_data_from": "2025-03-01",
    "training_data_to": "2026-02-28",
    "feature_list": ["attendance_rate_30d", ...],
    "evaluation_metrics": { "rmse": 6.3, "mae": 4.9, "r2": 0.81 },
    "promoted_to_production": true
}
```
