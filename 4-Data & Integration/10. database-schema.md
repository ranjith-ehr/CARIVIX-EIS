## Table: `employees`

Stores all employee account data.

```sql
CREATE TABLE employees (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name                VARCHAR(255) NOT NULL,
    email               VARCHAR(255) UNIQUE NOT NULL,
    password_hash       VARCHAR(255) NOT NULL,
    role                VARCHAR(20) NOT NULL CHECK (role IN ('employee', 'manager', 'admin')),
    department          VARCHAR(100),
    manager_id          UUID REFERENCES employees(id) ON DELETE SET NULL,
    timezone            VARCHAR(50) NOT NULL DEFAULT 'UTC',
    notification_prefs  JSONB DEFAULT '{}',
    is_active           BOOLEAN NOT NULL DEFAULT TRUE,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deactivated_at      TIMESTAMPTZ
);
```

| Column | Notes |
|---|---|
| `id` | UUID primary key, auto-generated |
| `password_hash` | bcrypt hash (cost factor 12); plaintext never stored |
| `role` | Enum-constrained: `employee`, `manager`, `admin` |
| `manager_id` | Self-referential FK; NULL for top-level managers and admins |
| `notification_prefs` | JSONB object; keys: `email`, `push`, `in_app` (booleans) |
| `is_active` | Set to FALSE on account deactivation; records are never hard-deleted |

**Indexes:**
```sql
CREATE INDEX idx_employees_email ON employees(email);
CREATE INDEX idx_employees_manager_id ON employees(manager_id);
CREATE INDEX idx_employees_department ON employees(department);
```

## Table: `attendance_logs`

One record per employee check-in session.

```sql
CREATE TABLE attendance_logs (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    employee_id         UUID NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
    login_time          TIMESTAMPTZ NOT NULL,
    logout_time         TIMESTAMPTZ,
    work_hours          NUMERIC(5,2),               -- computed on checkout
    status              VARCHAR(20) CHECK (status IN ('present', 'late', 'half_day', 'absent', 'incomplete')),
    is_late             BOOLEAN NOT NULL DEFAULT FALSE,
    gis_validated       BOOLEAN NOT NULL DEFAULT FALSE,
    latitude            NUMERIC(10,7),
    longitude           NUMERIC(10,7),
    correction_note     TEXT,                       -- populated on manual corrections
    corrected_by        UUID REFERENCES employees(id) ON DELETE SET NULL,
    data_quality_flag   VARCHAR(30) DEFAULT 'clean',
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Notes |
|---|---|
| `work_hours` | Calculated as `(logout_time - login_time)` in hours; NULL until checkout |
| `status` | Assigned by pipeline transformation; can be overridden by manual correction |
| `correction_note` | Required when `corrected_by` is not null |
| `data_quality_flag` | Set by data pipeline; values: `clean`, `incomplete`, `timestamp_error`, `duplicate`, `zero_duration`, `gps_invalid`, `future_dated` |

**Indexes:**
```sql
CREATE INDEX idx_attendance_employee_id ON attendance_logs(employee_id);
CREATE INDEX idx_attendance_login_time ON attendance_logs(login_time);
CREATE INDEX idx_attendance_employee_date ON attendance_logs(employee_id, DATE(login_time));
CREATE INDEX idx_attendance_status ON attendance_logs(status);
```

**Constraint:**
```sql
ALTER TABLE attendance_logs
ADD CONSTRAINT chk_logout_after_login
CHECK (logout_time IS NULL OR logout_time > login_time);
```

## Table: `performance_scores`

One record per employee per scoring date (daily recalculation).

```sql
CREATE TABLE performance_scores (
    id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    employee_id             UUID NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
    score_date              DATE NOT NULL,
    score                   NUMERIC(5,2) NOT NULL CHECK (score BETWEEN 0 AND 100),
    classification          VARCHAR(20) CHECK (classification IN ('high_performer', 'average', 'at_risk')),
    attendance_rate         NUMERIC(5,2),
    avg_work_hours          NUMERIC(5,2),
    late_login_count        INTEGER,
    idle_ratio              NUMERIC(4,3),
    overtime_frequency      NUMERIC(4,3),
    shap_summary            TEXT,
    model_version           VARCHAR(20),
    created_at              TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (employee_id, score_date)
);
```

| Column | Notes |
|---|---|
| `score_date` | Date for which the score was calculated (not the calculation run date) |
| `classification` | Assigned by Logistic Regression model |
| `shap_summary` | Plain-language explanation string generated from SHAP values |
| `model_version` | Version string of the XGBoost model used (e.g., `xgb_v3`) |

**Indexes:**
```sql
CREATE INDEX idx_scores_employee_date ON performance_scores(employee_id, score_date DESC);
CREATE INDEX idx_scores_classification ON performance_scores(classification);
```

## Table: `task_logs`

Employee task entries logged during sessions.

```sql
CREATE TABLE task_logs (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    employee_id         UUID NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
    session_id          UUID REFERENCES attendance_logs(id) ON DELETE SET NULL,

    logged_at           TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    data_quality_flag   VARCHAR(30) DEFAULT 'clean',
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**Indexes:**
```sql
CREATE INDEX idx_tasks_employee_id ON task_logs(employee_id);
CREATE INDEX idx_tasks_logged_at ON task_logs(logged_at);
```

## Table: `anomaly_flags`

Anomaly alerts generated by the Isolation Forest model.

```sql
CREATE TABLE anomaly_flags (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    employee_id         UUID NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
    attendance_log_id   UUID REFERENCES attendance_logs(id) ON DELETE SET NULL,
    anomaly_type        VARCHAR(50) NOT NULL,
    severity            VARCHAR(10) NOT NULL CHECK (severity IN ('low', 'medium', 'high')),
    anomaly_score       NUMERIC(5,4),               -- Isolation Forest score
    flag_date           DATE NOT NULL,
    status              VARCHAR(20) NOT NULL DEFAULT 'open'
                        CHECK (status IN ('open', 'acknowledged', 'escalated', 'dismissed')),
    resolved_by         UUID REFERENCES employees(id) ON DELETE SET NULL,
    resolution_note     TEXT,
    resolved_at         TIMESTAMPTZ,
    model_version       VARCHAR(20),
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**Indexes:**
```sql
CREATE INDEX idx_anomaly_employee_id ON anomaly_flags(employee_id);
CREATE INDEX idx_anomaly_status ON anomaly_flags(status);
CREATE INDEX idx_anomaly_flag_date ON anomaly_flags(flag_date DESC);
```

## Table: `audit_logs`

Immutable record of all significant system actions.

```sql
CREATE TABLE audit_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    actor_id        UUID REFERENCES employees(id) ON DELETE SET NULL,
    action_type     VARCHAR(50) NOT NULL,
    module          VARCHAR(50) NOT NULL,
    target_id       UUID,                           -- ID of the affected resource
    target_type     VARCHAR(50),                    -- e.g., 'attendance_log', 'employee'
    old_value       JSONB,
    new_value       JSONB,
    ip_address      INET,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

> Audit log records are **never updated or deleted** during the 12-month retention period. Deletion is performed by a scheduled archival job, not manually.

**Indexes:**
```sql
CREATE INDEX idx_audit_actor_id ON audit_logs(actor_id);
CREATE INDEX idx_audit_created_at ON audit_logs(created_at DESC);
CREATE INDEX idx_audit_action_type ON audit_logs(action_type);
```

## Redis — Cache Layer

**Host:** 
**Connection string:** `redis://<memorystore-ip>:6379`

### Key Patterns and TTL

| Key Pattern | Value | TTL | Purpose |
|---|---|---|---|
| `session:{user_id}` | JWT token string | 28800s (8h) | Active session token |
| `score:{employee_id}` | JSON performance score object | 3600s (1h) | Cached performance score |
| `attendance:{employee_id}:{date}` | JSON attendance record | 900s (15m) | Today's attendance record cache |
| `team:{manager_id}:summary` | JSON team summary | 300s (5m) | Manager dashboard summary |
| `ml:classification:{employee_id}` | Classification string | 86400s (24h) | Cached employee classification |

### Cache Invalidation

| Event | Keys Invalidated |
|---|---|
| Employee checks in or out | `attendance:{employee_id}:{date}`, `team:{manager_id}:summary` |
| Performance score recalculated | `score:{employee_id}` |
| Employee classification updated | `ml:classification:{employee_id}` |

Cache invalidation is triggered by the relevant Flask blueprint via a `cache.delete(key)` call after the corresponding database write.

## Migration Versioning

Migrations are managed using **Alembic**. Migration files are stored in `db/migrations/`.

```bash
# Apply all pending migrations
alembic upgrade head

# Rollback one migration
alembic downgrade -1

# Check current revision
alembic current
```

All migration files follow the naming convention: `{revision_id}_{short_description}.py`
