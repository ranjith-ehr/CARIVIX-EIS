# API Reference

This document describes all backend API endpoints for the Attendance Tracking system, organized by blueprint module. The API follows REST conventions and uses JSON for all request and response bodies.

**Base URL:** `https://api.attendancetracker.yourdomain.com/v1`

---

## Authentication

All endpoints require a valid JWT Bearer token unless otherwise noted.

### Obtaining a Token

```
POST /auth/login
```

**Request Body:**

```json
{
  "email": "user@example.com",
  "password": "your_password"
}
```

**Response `200 OK`:**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 28800
}
```

**Response `401 Unauthorized`:**

```json
{
  "error": "INVALID_CREDENTIALS",
  "message": "Email or password is incorrect."
}
```

### Using the Token

Include the token in the `Authorization` header for all subsequent requests:

```
Authorization: Bearer <access_token>
```

### Token Expiry and Refresh

Tokens expire after **8 hours**. Use the refresh endpoint to obtain a new token without re-authenticating:

```
POST /auth/refresh
Authorization: Bearer <access_token>
```

**Response `200 OK`:**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 28800
}
```

---

## Rate Limiting

| Limit | Value |
|---|---|
| Requests per minute (per user) | 60 |
| Requests per minute (per IP) | 200 |

Rate limit headers returned on all responses:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 58
X-RateLimit-Reset: 1712825400
```

When the limit is exceeded, the API returns `429 Too Many Requests`.

---

## Error Response Format

All error responses follow a consistent envelope:

```json
{
  "error": "ERROR_CODE",
  "message": "Human-readable description of the error.",
  "details": {}
}
```

### Common HTTP Status Codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `201` | Resource created |
| `400` | Bad request — invalid input |
| `401` | Unauthorized — missing or invalid token |
| `403` | Forbidden — insufficient role permissions |
| `404` | Resource not found |
| `422` | Unprocessable entity — validation error |
| `429` | Too many requests |
| `500` | Internal server error |

## Session Blueprint (`session_bp`)

Handles authentication, session management, and user account operations.

---

### POST `/auth/login`
Authenticate a user and receive a JWT token.

**Auth required:** No

**Request Body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `email` | string | Yes | Registered email address |
| `password` | string | Yes | Account password |

**Responses:**

| Code | Description |
|---|---|
| `200` | Login successful; returns `access_token` |
| `401` | Invalid credentials |
| `422` | Missing required fields |

---

### POST `/auth/logout`
Invalidate the current session token.

**Auth required:** Yes

**Request Body:** None

**Responses:**

| Code | Description |
|---|---|
| `200` | Session invalidated |
| `401` | Token already expired or invalid |

---

### POST `/auth/refresh`
Obtain a new access token using an existing valid token.

**Auth required:** Yes

**Responses:**

| Code | Description |
|---|---|
| `200` | Returns new `access_token` |
| `401` | Token expired or invalid |

---

### POST `/auth/reset-password`
Request a password reset email.

**Auth required:** No

**Request Body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `email` | string | Yes | Registered email address |

**Responses:**

| Code | Description |
|---|---|
| `200` | Reset email sent (returned regardless of whether the email exists, for security) |
| `422` | Invalid email format |

---

### GET `/users/me`
Return the profile of the currently authenticated user.

**Auth required:** Yes

**Response `200 OK`:**

```json
{
  "id": "usr_a1b2c3",
  "name": "Jane Smith",
  "email": "jane.smith@example.com",
  "role": "employee",
  "department": "Engineering",
  "manager_id": "usr_x9y8z7",
  "created_at": "2025-01-15T08:00:00Z"
}
```

---

### PATCH `/users/me`
Update the current user's profile fields.

**Auth required:** Yes

**Request Body (partial update supported):**

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | No | Full name |
| `notification_preferences` | object | No | Notification settings |

**Responses:**

| Code | Description |
|---|---|
| `200` | Profile updated |
| `422` | Validation error on submitted fields |

## Attendance Blueprint (`attendance_bp`)

Handles check-in, check-out, attendance records, and status queries.

---

### POST `/attendance/checkin`
Record a check-in for the authenticated employee.

**Auth required:** Yes

**Request Body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `latitude` | float | Yes | GPS latitude at time of check-in |
| `longitude` | float | Yes | GPS longitude at time of check-in |
| `face_token` | string | Yes | Face verification token from the face verification service |

**Response `201 Created`:**

```json
{
  "record_id": "att_001abc",
  "employee_id": "usr_a1b2c3",
  "login_time": "2026-04-11T09:04:22Z",
  "status": "present",
  "gis_validated": true,
  "face_verified": true
}
```

**Responses:**

| Code | Description |
|---|---|
| `201` | Check-in recorded successfully |
| `400` | Already checked in today |
| `403` | Face verification failed |
| `422` | Missing or invalid GPS coordinates |
| `451` | GIS validation failed — location outside zone |

---

### POST `/attendance/checkout`
Record a check-out for the authenticated employee.

**Auth required:** Yes

**Request Body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `face_token` | string | Yes | Face verification token |

**Response `200 OK`:**

```json
{
  "record_id": "att_001abc",
  "logout_time": "2026-04-11T18:12:05Z",
  "work_hours": 9.13,
  "status": "present"
}
```

**Responses:**

| Code | Description |
|---|---|
| `200` | Check-out recorded |
| `400` | No active check-in found for today |
| `403` | Face verification failed |

---

### GET `/attendance/me`
Return the authenticated employee's attendance records.

**Auth required:** Yes

**Query Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `from` | date (YYYY-MM-DD) | No | Start date filter (default: first of current month) |
| `to` | date (YYYY-MM-DD) | No | End date filter (default: today) |
| `status` | string | No | Filter by status: `present`, `absent`, `late`, `half_day` |
| `page` | integer | No | Page number (default: 1) |
| `per_page` | integer | No | Results per page (default: 30, max: 100) |

**Response `200 OK`:**

```json
{
  "total": 22,
  "page": 1,
  "per_page": 30,
  "records": [
    {
      "record_id": "att_001abc",
      "date": "2026-04-11",
      "login_time": "2026-04-11T09:04:22Z",
      "logout_time": "2026-04-11T18:12:05Z",
      "work_hours": 9.13,
      "status": "present",
      "late": false,
      "gis_validated": true,
      "face_verified": true
    }
  ]
}
```

---

### GET `/attendance/team`
Return attendance records for the manager's team.

**Auth required:** Yes | **Roles:** Manager, Admin

**Query Parameters:** Same as `/attendance/me`, plus:

| Parameter | Type | Required | Description |
|---|---|---|---|
| `employee_id` | string | No | Filter to a specific employee |
| `department` | string | No | Filter by department name (Admin only) |

**Responses:**

| Code | Description |
|---|---|
| `200` | Returns paginated team attendance records |
| `403` | Caller does not have Manager or Admin role |

---

### PATCH `/attendance/{record_id}`
Manually correct an attendance record.

**Auth required:** Yes | **Roles:** Manager, Admin

**Path Parameter:** `record_id` — the ID of the attendance record to correct.

**Request Body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `login_time` | datetime | No | Corrected login timestamp (ISO 8601) |
| `logout_time` | datetime | No | Corrected logout timestamp (ISO 8601) |
| `status` | string | No | Override status: `present`, `absent`, `late`, `half_day` |
| `correction_reason` | string | Yes | Required audit note explaining the correction |

**Responses:**

| Code | Description |
|---|---|
| `200` | Record updated; correction logged in audit trail |
| `403` | Insufficient role |
| `404` | Record not found |
| `422` | Validation error (e.g., logout before login) |

## Task Blueprint (`task_bp`)

Handles employee task logs and activity recording.

---

### GET `/tasks/me`
Return the authenticated employee's task log entries.

**Auth required:** Yes

**Query Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `from` | date | No | Start date |
| `to` | date | No | End date |

```json
  "total": 14,
  "records": [
    {
      "task_id": "tsk_00123",
      "date": "2026-04-11",

      "duration_minutes": 90
    }
  ]
}
```

---

### POST `/tasks`
Log a new task entry for the authenticated employee.

**Auth required:** Yes

**Request Body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `description` | string | Yes | Task description (max 500 characters) |
| `duration_minutes` | integer | Yes | Time spent on the task in minutes |
| `logged_at` | datetime | No | Timestamp of task completion (defaults to now) |

**Responses:**

| Code | Description |
|---|---|
| `201` | Task logged |
| `422` | Missing required fields or validation error |

**Request Body:**
