# CI/CD Pipeline

This document describes the full CI/CD workflow — from code push through automated testing, Docker image build, Artifact Registry storage, Terraform provisioning, and Cloud Run deployment.

---

## Pipeline Overview

```
Developer pushes to GitHub
         ↓
GitHub Actions Workflow triggered
         ↓
┌─────────────────────────────────┐
│  Stage 1: Test Gate             │
│  - Unit tests (pytest)          │
│  - Integration tests            │
│  - Linting (flake8, black)      │
│  - ML model tests               │
│  → Must pass to proceed         │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│  Stage 2: Docker Build          │
│  - Build api image              │
│  - Build pipeline image         │
│  - Tag with Git SHA + version   │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│  Stage 3: Push to Registry      │
│  - Push to Artifact Registry    │
│  - Tag :latest on main branch   │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│  Stage 4: Terraform Plan/Apply  │
│  - terraform plan on PR         │
│  - terraform apply on main      │
│  → Only infra changes applied   │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│  Stage 5: Deploy to Cloud Run   │
│  - gcloud run deploy            │
│  - Smoke test on new revision   │
│  → Traffic routed on success    │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│  Stage 6: Post-Deploy           │
│  - Notify Slack #deployments    │
│  - Update Cloud Monitoring      │
│    deployment annotation        │
└─────────────────────────────────┘
```

---

## Branch Strategy

| Branch | Trigger | Environment |
|---|---|---|
| `feature/*` | Push → runs tests only, no deploy | — |
| `develop` | Push → deploys to `staging` environment | staging |
| `main` | Push → deploys to `production` | production |
| Pull Request → `main` | Opens → runs tests + terraform plan | — |

## GitHub Actions Workflow

### Main Workflow File

`.github/workflows/deploy.yml`

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

env:
  PROJECT_ID: attendance-tracker-prod
  REGION: asia-south1
  REGISTRY: asia-south1-docker.pkg.dev
  REPO: attendance-tracker-prod/attendance
  API_IMAGE: api
  PIPELINE_IMAGE: pipeline

jobs:
  test:
    name: Test Gate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: |
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-dev.txt

      - name: Run linting
        run: |
          flake8 backend/ --max-line-length=100
          black backend/ --check

      - name: Run unit tests
        run: pytest backend/tests/unit/ -v --cov=backend/app --cov-report=xml

      - name: Run integration tests
        run: pytest backend/tests/integration/ -v
        env:
          DATABASE_URL: ${{ secrets.TEST_DATABASE_URL }}
          REDIS_URL: redis://localhost:6379

      - name: Run ML model tests
        run: pytest ml/tests/ -v

      - name: Upload coverage
        uses: codecov/codecov-action@v4
        with:
          file: ./coverage.xml

  build:
    name: Build Docker Images
    runs-on: ubuntu-latest
    needs: test
    if: github.event_name == 'push'
    outputs:
      api_image: ${{ steps.meta.outputs.api_image }}
      pipeline_image: ${{ steps.meta.outputs.pipeline_image }}

    steps:
      - uses: actions/checkout@v4

      - name: Authenticate to GCP
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SA_KEY }}

      - name: Configure Docker
        run: gcloud auth configure-docker ${{ env.REGISTRY }}

      - name: Build and push API image
        run: |
          IMAGE=${{ env.REGISTRY }}/${{ env.REPO }}/${{ env.API_IMAGE }}
          docker build -t ${IMAGE}:${{ github.sha }} -t ${IMAGE}:latest ./backend/
          docker push ${IMAGE}:${{ github.sha }}
          docker push ${IMAGE}:latest

      - name: Build and push Pipeline image
        run: |
          IMAGE=${{ env.REGISTRY }}/${{ env.REPO }}/${{ env.PIPELINE_IMAGE }}
          docker build -t ${IMAGE}:${{ github.sha }} -t ${IMAGE}:latest ./pipeline/
          docker push ${IMAGE}:${{ github.sha }}
          docker push ${IMAGE}:latest

  terraform:
    name: Terraform
    runs-on: ubuntu-latest
    needs: build
    if: github.event_name == 'push'

    steps:
      - uses: actions/checkout@v4

      - name: Authenticate to GCP
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SA_KEY }}

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: 1.6.0

      - name: Terraform Init
        working-directory: infra/terraform
        run: terraform init -backend-config="bucket=attendance-tfstate-prod"

      - name: Terraform Plan
        working-directory: infra/terraform
        run: terraform plan -var-file="environments/prod.tfvars" -out=tfplan

      - name: Terraform Apply
        if: github.ref == 'refs/heads/main'
        working-directory: infra/terraform
        run: terraform apply -auto-approve tfplan

  deploy:
    name: Deploy to Cloud Run
    runs-on: ubuntu-latest
    needs: terraform
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'

    steps:
      - name: Authenticate to GCP
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SA_KEY }}

      - name: Deploy API to Cloud Run
        run: |
          gcloud run deploy attendance-api \
            --image=${{ env.REGISTRY }}/${{ env.REPO }}/${{ env.API_IMAGE }}:${{ github.sha }} \
            --region=${{ env.REGION }} \
            --project=${{ env.PROJECT_ID }} \
            --no-traffic  # deploy without routing traffic yet

      - name: Smoke test new revision
        run: |
          NEW_REVISION=$(gcloud run revisions list \
            --service=attendance-api \
            --region=${{ env.REGION }} \
            --sort-by=~CREATED \
            --limit=1 \
            --format="value(name)")

          REVISION_URL=$(gcloud run revisions describe $NEW_REVISION \
            --region=${{ env.REGION }} \
            --format="value(status.url)")

          STATUS=$(curl -s -o /dev/null -w "%{http_code}" "${REVISION_URL}/health")
          if [ "$STATUS" != "200" ]; then
            echo "Smoke test failed with status $STATUS"
            exit 1
          fi

      - name: Route traffic to new revision
        run: |
          gcloud run services update-traffic attendance-api \
            --region=${{ env.REGION }} \
            --to-latest

  notify:
    name: Post-Deploy Notification
    runs-on: ubuntu-latest
    needs: deploy
    if: always()

    steps:
      - name: Notify Slack
        uses: slackapi/slack-github-action@v1.26.0
        with:
          payload: |
            {
              "text": "${{ needs.deploy.result == 'success' && '✅' || '❌' }} Deployment to production ${{ needs.deploy.result }} — ${{ github.repository }}@${{ github.sha }}"
            }
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_DEPLOY_WEBHOOK }}
```

## Required GitHub Secrets

| Secret | Description |
|---|---|
| `GCP_SA_KEY` | JSON key for the CI/CD service account |
| `TEST_DATABASE_URL` | PostgreSQL connection string for the test database |
| `SLACK_DEPLOY_WEBHOOK` | Slack incoming webhook URL for deployment notifications |

## Docker Image Tagging Convention

| Tag | When Applied | Used For |
|---|---|---|
| `:latest` | Every push to `main` | Default image for Cloud Run |
| `:{git-sha}` | Every push | Precise version tracking; used in rollback |
| `:v{semver}` | On tagged release (e.g., `v1.2.0`) | Release artifacts |

## Rollback Procedure

### Rollback to Previous Image

```bash
# List recent revisions
gcloud run revisions list \
  --service=attendance-api \
  --region=asia-south1 \
  --sort-by=~CREATED \
  --limit=5

# Route 100% traffic to the previous revision
gcloud run services update-traffic attendance-api \
  --region=asia-south1 \
  --to-revisions=attendance-api-00041-abc=100
```

### Emergency Rollback (No Pipeline)

```bash
# Deploy a specific image directly
gcloud run deploy attendance-api \
  --image=asia-south1-docker.pkg.dev/attendance-tracker-prod/attendance/api:{previous-sha} \
  --region=asia-south1
```

## Test Gate Requirements

The following must pass before any build proceeds:

| Check | Tool | Failure Action |
|---|---|---|
| Python linting | flake8 + black | Block build |
| Unit tests | pytest | Block build |
| Integration tests | pytest | Block build |
| ML model tests | pytest | Block build |
| Code coverage | codecov | Warn only (< 70% triggers Slack alert) |
