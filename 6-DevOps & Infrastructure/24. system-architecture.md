# System Architecture

This document covers the full end-to-end architecture of the Attendance Tracking system — from the client layer through the API backend, data pipeline, ML layer, and storage. It includes the request lifecycle for a check-in, blueprint routing structure, and WTForms validation.

## Layer Breakdown

### Client Layer

| Client | Tech Stack | Communication |
|---|---|---|
| Web App | Browser (React or vanilla JS) | HTTPS REST to Flask backend |
| Mobile App | React Native (Expo), TypeScript | HTTPS REST + Socket.io WebSocket |

The mobile app uses Socket.io for real-time push events (anomaly alerts, check-in confirmations). The web app polls for notifications.

### Flask Backend (Cloud Run)

The backend is a Flask application served by Gunicorn with a WSGI interface, containerized and deployed on Cloud Run.

**Gunicorn config:**
```
workers = 4
worker_class = gthread
threads = 2
timeout = 30
bind = 0.0.0.0:8080
```

**Middleware stack:**
1. TLS termination (Cloud Run handles)
2. JWT authentication middleware (`flask_jwt_extended`)
3. Request logging middleware
4. WTForms validation (per blueprint)
5. SQLAlchemy ORM
6. Redis session cache

## Blueprint Routing Structure

```
/auth/           → session_bp       (login, logout, refresh, password reset)
/users/          → session_bp       (profile read/update)
/attendance/     → attendance_bp    (check-in, check-out, records, corrections)
/tasks/          → task_bp          (task log CRUD)
/activity/       → activity_bp      (heartbeat, idle time)
/performance/    → performance_bp   (score read, team scores)
/alerts/         → alerts_bp        (anomaly flag read/update)
/reports/        → reports_bp       (report generation, scheduling, export)
/admin/          → admin_bp         (user mgmt, geofence, rules, ML schedule)
/chatbot/        → chatbot_bp       (query intake, response routing)
```

### Blueprint Registration

```python
# app/__init__.py
from app.blueprints.session import session_bp
from app.blueprints.attendance import attendance_bp
from app.blueprints.task import task_bp
from app.blueprints.activity import activity_bp

def create_app(config_name='production'):
    app = Flask(__name__)
    app.config.from_object(config[config_name])

    db.init_app(app)
    jwt.init_app(app)
    cache.init_app(app)

    app.register_blueprint(session_bp,    url_prefix='/auth')
    app.register_blueprint(attendance_bp, url_prefix='/attendance')
    app.register_blueprint(task_bp,       url_prefix='/tasks')
    app.register_blueprint(activity_bp,   url_prefix='/activity')
    # ... additional blueprints

    return app
```

## WTForms Validation Layer

WTForms validates incoming request data before it reaches business logic.

### Forms

```python
# app/forms/attendance_forms.py
from wtforms import Form, FloatField, StringField, validators

class AttendanceForm(Form):
    latitude    = FloatField('Latitude',   [validators.NumberRange(min=-90, max=90)])
    longitude   = FloatField('Longitude',  [validators.NumberRange(min=-180, max=180)])
    face_token  = StringField('FaceToken', [validators.DataRequired(), validators.Length(min=32)])

class SessionForm(Form):
    email       = StringField('Email',    [validators.DataRequired(), validators.Email()])
    password    = StringField('Password', [validators.DataRequired(), validators.Length(min=8)])

class TaskForm(Form):
    description      = StringField('Description',    [validators.DataRequired(), validators.Length(max=500)])
    duration_minutes = IntegerField('Duration',      [validators.NumberRange(min=1, max=720)])
```

### Usage in Blueprint

```python
@attendance_bp.route('/checkin', methods=['POST'])
@jwt_required()
def checkin():
    form = AttendanceForm(data=request.get_json())
    if not form.validate():
        return jsonify({'error': 'VALIDATION_ERROR', 'details': form.errors}), 422
    # proceed with business logic
```

## Request Lifecycle — Employee Check-In

End-to-end flow for `POST /attendance/checkin`:

```
1. Mobile app sends POST /attendance/checkin
   Body: { latitude, longitude, face_token }
   Header: Authorization: Bearer <jwt_token>
        ↓
2. JWT middleware validates token
   → Extracts employee_id, role
   → Checks Redis for session: session:{employee_id}
        ↓
3. WTForms AttendanceForm validates body
   → Checks lat/lng ranges, face_token format
        ↓
4. attendance_bp.checkin() business logic:
   a. Check no existing check-in today → attendance_logs query
   b. Call Face Recognition Service (HTTP POST to GPU VM):
      POST http://face-recognition-vm:5001/verify
      Body: { employee_id, face_token }
      → Returns: { verified: true, confidence: 0.94 }
   c. Call GIS Validation (in-process):
      → Query geofence_zones for employee's assigned zones
      → Point-in-polygon check (latitude, longitude)
      → Returns: { validated: true, zone_id: "..." }
   d. If both pass → INSERT into attendance_logs
   e. Invalidate Redis cache: attendance:{employee_id}:{date}
   f. Invalidate team summary cache: team:{manager_id}:summary
        ↓
5. Return 201 with attendance record
        ↓
6. (Async) Pipeline ingestion picks up new record within 15 minutes
7. (Async) Anomaly detection model scores the record in nightly run
```

## Inter-Service Communication

| From | To | Protocol | Notes |
|---|---|---|---|
| Flask Backend | Face Recognition VM | HTTP REST (internal VPC) | Synchronous; 3s timeout |
| Flask Backend | LLM Inference VM | HTTP REST (internal VPC) | Async; 8s timeout |
| Flask Backend | PostgreSQL (Cloud SQL) | TCP via Cloud SQL Auth Proxy | SQLAlchemy connection pool |
| Flask Backend | Redis (Memorystore) | TCP | `flask-caching` with Redis backend |
| Mobile App | Flask Backend | HTTPS + Socket.io | WebSocket for real-time notifications |
| Pipeline Jobs | GCS | GCS SDK | Parquet read/write |
| ML Jobs | BigQuery ML | BigQuery API | Batch inference jobs |
